# ШАГ 1: Категоризация и приоритетность

| Tier | Категория | Проекты | Обоснование |
|------|-----------|---------|-------------|
| **Tier-1** | Enterprise & SaaS | LMS «Lucy Nails», ERP «LOMuebles», CRM «Я и Ты» | Сложные Full-Stack системы с business-logic, ролевой моделью, интеграциями и продуктовой глубиной |
| **Tier-2** | AI Ecosystem & Automation | Экосистема AI-автоматизации контента (6 проектов объединены) | Уникальная инженерная экосистема на стыке AI, browser automation и data pipelines |
| **Tier-3** | Commercial Websites | Сайт LOMuebles, Стоматология Prozub.by | Коммерческие сайты с конверсионной архитектурой и SEO-оптимизацией |

### Логика расставления приоритетов

- **Tier-1** демонстрирует способность проектировать и реализовывать сложные бизнес-системы «от нуля до продакшена» — именно это ценят работодатели и заказчики уровня Enterprise.
- **Tier-2** выделяет уникальную компетенцию: создание AI-инфраструктуры, работа с LLM (GPT-4, Claude, Gemini), построение data pipelines и browser automation. Это дифференциатор, которого нет у 95% разработчиков.
- **Tier-3** подтверждает умение создавать коммерчески эффективные продукты, ориентированные на конверсию.

### 🎓 LMS «Lucy Nails» — Платформа онлайн-обучения

> Полноценная образовательная платформа для продажи и прохождения видеокурсов со встроенной аналитикой и управлением контентом.

**Роль:** Full-Stack разработчик (Архитектура, Backend, Frontend, интеграции сторонних сервисов)

**Бизнес-задача:**
Требовалось создать надежную площадку для инфобизнеса, которая автоматизирует продажу курсов, защитит видео от пиратства и предоставит удобный личный кабинет для студентов с доступом через Telegram.

**Техническая реализация:**

| Задача | Решение | Почему это сложно |
|--------|---------|-------------------|
| Защита видеоконтента | Интеграция с **Kinescope API** — видео не хранятся на своём сервере, стриминг через зашифрованные DRM-потоки | Прямая ссылка на видео недоступна, невозможно скачать или дать доступ без авторизации |
| Безшовная авторизация | **Telegram Login Widget** + fallback на классический логин/пароль | Нужно было связать Telegram ID с внутренним user_id, поддержать оба потока авторизации и обеспечить безопасность сессий |
| Управление контентом | Админ-панель с CRUD для курсов → модулей → уроков (3-уровневая иерархия), загрузка медиафайлов, ручное управление доступом | Гранулярный контроль: доступ выдаётся не ко всей платформе, а к конкретным курсам для конкретных пользователей |
| Прогресс студента | Серверный трекинг прохождения каждого урока, расчёт % завершения модуля/курса | Необходимо обеспечить идемпотентность при множественных запросах и корректный rollback |
| Аналитика продаж | Дашборд с метриками: выручка, количество покупок, активные пользователи | Агрегации по временным периодам с учётом транзакционной целостности |

**Стек технологий:**

| Слой | Технологии |
|------|------------|
| **Backend** | Python, FastAPI, SQLAlchemy, Alembic |
| **Frontend** | Next.js (React), TypeScript, Tailwind CSS, shadcn/ui |
| **DB** | PostgreSQL |
| **Интеграции** | Kinescope API, Telegram Bot API |
| **DevOps** | Docker, seed-скрипты для автоматического развёртывания |

🔗 **Продакшен:** [lucysmirnova.ru](https://lucysmirnova.ru/)

### 🏭 ERP «LOMuebles» — Система управления мебельным производством

> Комплексная система автоматизации мебельного предприятия, объединяющая продажи, Канбан-доску производства, складской учет и финансовую отчетность.

**Роль:** Full-Stack & BaaS разработчик

**Бизнес-задача:**
Устранить хаос в операционных процессах мебельной фабрики, обеспечив «прозрачность» на всех этапах: от поступления лида до контроля остатков на складе и расчета маржинальности заказов. Мебельное производство в Испании управлялось через Excel-таблицы и мессенджеры — это приводило к потере лидов, отсутствию контроля над складскими остатками и невозможности рассчитать реальную маржинальность.

**Техническая реализация:**

| Задача | Решение | Почему это сложно |
|--------|---------|-------------------|
| CRM-модуль | Управление лидами, контактами, компаниями с историей взаимодействий. Входящие заявки через **вебхук Tilda** | Нужна гибкая модель данных для разных типов контрагентов и автоматическая классификация лидов |
| Канбан производства | Визуализация заказов по стадиям (drag-and-drop). Задачи привязываются к заказам, клиентам, проектам | Сложная связь many-to-many между сущностями с сохранением порядка отображения |
| Складской учёт | Номенклатура → Локации → Движение товаров → Бронирование под заказ → План vs Факт | Двойная бухгалтерия: плановый расход по смете vs реальное списание. Алерты по критическим остаткам |
| Финансовая аналитика | Автоматический расчёт KPI, маржинальности каждого заказа, P&L всего предприятия | Вся бизнес-логика (RLS, агрегации, расчёт маржи) реализована **на уровне БД** через SQL-функции в Supabase |
| Ролевая модель | Разграничение прав: админ, менеджер, специалист, кладовщик | Row Level Security (RLS) в Supabase — каждый видит только «свои» данные |

**Стек технологий:**

| Слой | Технологии |
|------|------------|
| **Frontend** | React, TypeScript, Vite, Tailwind CSS, shadcn/ui |
| **Backend & DB** | Supabase (PostgreSQL), SQL-функции, RLS-политики, миграции |
| **State Management** | Кастомные React Hooks (per-entity) |
| **Интеграции** | Tilda Webhooks |

### 💇 CRM «Я и Ты» — Система управления сервисным бизнесом

> CRM-платформа для управления записями, складом и финансами салонов красоты и частных мастеров.

**Роль:** Full-Stack разработчик (API-дизайн, Frontend SPA, архитектура БД и DevOps)

**Бизнес-задача:**
Автоматизировать работу мастера/администратора: внедрить онлайн-запись для клиентов, автоматическое списание расходных материалов, ведение клиентской базы и прозрачный финансовый учет с расчетом налогов. Клиенты записывались через WhatsApp-переписку, что приводило к конфликтам расписания и потерянным записям.

**Техническая реализация:**

| Задача | Решение | Почему это сложно |
|--------|---------|-------------------|
| Интерактивный календарь | Сетка записей с drag-and-drop, публичная страница самозаписи клиентов | Конфликт-резолюция: система предотвращает двойные бронирования и учитывает длительность процедур |
| Складской учёт | Автоматическое списание расходных материалов при оказании услуги, контроль остатков, алерты при минимуме | Связка «услуга → набор расходников → автосписание» требует атомарных транзакций |
| Финансовый модуль | Учёт доходов/расходов, разные методы оплаты, расчёт налогов по профилю | Формулы налогообложения зависят от настроек профиля (ИП, самозанятый, процент) |
| Аналитика | Графики: выручка по периодам, популярность услуг, активность клиентов, LTV | Агрегации с фильтрацией по множеству измерений |
| Мгновенные уведомления | Telegram-бот на aiogram 3: пуш о новых записях, изменениях, напоминания | Реал-тайм нотификации с retry-логикой и форматированием под Telegram Markdown |

**Стек технологий:**

| Слой | Технологии |
|------|------------|
| **Backend** | Python, Django, Django REST Framework |
| **Frontend** | Vue.js 3, Vite, Pinia |
| **DB** | PostgreSQL |
| **Bot** | aiogram 3 |
| **DevOps** | Docker, автоматический бэкап в облако |

### 🤖 Экосистема AI-автоматизации контента

> Единая инфраструктура из 5 микросервисов для поиска трендов, генерации, разметки и массовой выгрузки визуального контента с помощью AI (GPT-4, Claude, Midjourney).

**Роль:** AI Engineer, Backend & Desktop Developer (Разработка LLM-взаимодействий, парсеров, серверов и нативных десктопных GUI)

**Бизнес-задача:**
Полностью автоматизировать рутину стоковых фотографов и контент-мейкеров — от исследования рынка и предсказания прибыльных ниш до автоматической генерации метаданных (Keywords, Title) и внедрения их в EXIF тысяч медиа-файлов, превратив нейронный хаос в монетизируемый системный конвейер.

#### Компоненты экосистемы

| # | Компонент | Тип продукта | Назначение |
|---|-----------|-------------|------------|
| 1 | **IQStocker Bot** | Telegram Bot + Web Admin | AI-аналитика продаж, прогнозирование, рекомендации трендовых ниш |
| 2 | **IQStocker Generating** | Web App + Chrome Extension | SaaS-платформа для автоматизации генерации через Midjourney с лицензированием |
| 3 | **Adobe Scrapper** | Desktop-утилита | Парсинг и массовое скачивание референсов с Adobe Stock |
| 4 | **GPT-4 Vision Pipeline** | CLI-сервис | Мультипоточный анализ изображений через GPT Vision + генерация промптов для Midjourney |
| 5 | **GPT Video Vision** | CLI-сервис | Пакетная разметка видео: извлечение кадров → AI-анализ → запись метаданных через ffmpeg |
| 6 | **GPT-Vision-Batch** | Desktop GUI (PySide6) | Массовое заполнение EXIF-метаданных изображений через GPT-4 Vision с Batch API |

#### Стек технологий (сводный по экосистеме)

| Слой | Технологии |
|------|------------|
| **AI/LLM** | OpenAI GPT-4 Vision, Anthropic Claude, Google Gemini, Batch API |
| **Backend** | Python, FastAPI, SQLAlchemy, Alembic |
| **Frontend** | Next.js, Tailwind CSS, Jinja2 (admin) |
| **Bot** | aiogram 3 (FSM, Webhooks) |
| **Browser Automation** | Chrome Extension (TypeScript), Selenium/Playwright |
| **Desktop GUI** | PySide6 (Qt for Python) |
| **DB** | Supabase (PostgreSQL), Redis |
| **Инструменты** | ffmpeg, ExifTool, piexif |
| **DevOps** | Docker, Railway |
| **Платёжные системы** | Boosty, Tribute (через Webhooks) |

🔗 **Продакшен генератора:** [gen.iqstocker.com](https://gen.iqstocker.com)

| Задача | Решение | Почему это сложно |

# ШАГ 3: Архитектура объединённого кейса (Tier-2)

## Единая архитектурная картина экосистемы

Экосистема AI-автоматизации контента — это не набор разрозненных утилит, а **связанная система**, в которой каждый компонент выполняет свою роль в едином конвейере создания и продажи стокового контента.

### Архитектурные слои

```
┌─────────────────────────────────────────────────────────────────────┐
│                     📊 АНАЛИТИЧЕСКИЙ СЛОЙ                          │
│  IQStocker Bot (Telegram) + Admin Panel (FastAPI + Jinja2)         │
│  → Приём CSV-отчётов → AI-анализ (GPT-4/Claude/Gemini)            │
│  → Прогнозы продаж → Рекомендации трендов → Стратегия контента    │
├─────────────────────────────────────────────────────────────────────┤
│                     🔍 СЛОЙ ИССЛЕДОВАНИЯ                           │
│  Adobe Scrapper (Selenium + PySide6)                               │
│  → Парсинг Adobe Stock → Фильтрация по нише → Скачивание          │
│  → Базы референсных изображений для генерации                      │
├─────────────────────────────────────────────────────────────────────┤
│                     🎨 СЛОЙ ГЕНЕРАЦИИ                              │
│  IQStocker Generating (Web SaaS + Chrome Extension)                │
│  → Лицензирование → Автоматизация Midjourney → Очередь задач      │
│  → Массовая генерация по промптам → Скачивание результатов         │
├─────────────────────────────────────────────────────────────────────┤
│                     👁 СЛОЙ AI-ОБРАБОТКИ                           │
│  GPT-4 Vision Pipeline + GPT Video Vision                          │
│  → Мультипоточный анализ → Генерация промптов/описаний             │
│  → Запись метаданных (ffmpeg для видео)                             │
├─────────────────────────────────────────────────────────────────────┤
│                     📦 СЛОЙ ПОДГОТОВКИ К ПРОДАЖЕ                   │
│  GPT-Vision-Batch (PySide6 GUI)                                    │
│  → Batch API (−50% стоимости) → EXIF-запись (piexif/ExifTool)      │
│  → CSV-экспорт → Готовые файлы для загрузки на стоки               │
└─────────────────────────────────────────────────────────────────────┘
```

### Жизненный цикл данных в экосистеме

```
  ┌──────────────┐
  │  CSV-отчёты  │  Контент-мейкер загружает отчёты о продажах
  │  с фотостоков │  в Telegram-бот IQStocker
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │  AI-АНАЛИЗ   │  GPT-4 / Claude / Gemini анализируют:
  │  ПРОДАЖ      │  • Какие темы продаются лучше всего
  │              │  • Какие ниши на подъёме
```

### Детализация конвейера: как это работает на практике

#### 1. Этап Аналитики и Прогнозирования (Бот IQStocker)

Цикл стартует в Telegram-боте (aiogram 3 + Redis). Клиент загружает CSV-архивы реальных продаж. Модуль парсинга очищает данные, после чего кластер Core AI (мультимодальное агрегирование GPT-4, Claude и Gemini) сопоставляет цифры продаж пользователя с мировыми трендами. На выходе формируется прогнозная стратегия: какие ниши принесут наибольшую прибыль в следующем месяце. Все аналитические выкладки сохраняются в Supabase.

#### 2. Этап Сбора Базы Данных (Adobe Scrapper)

Для подготовки референсов запускается локальный Scrapper. Алгоритм, имитирующий поведение человека, массово прогоняет поисковые запросы по стокам (с фильтрами ориентации кадра, наличия людей, качества) и скачивает сотни тысяч миниатюр.

#### 3. Этап Генерации и Контроля (AI Pipeline & Chrome Extension)

Централизованное B2B/B2C решение через веб-платформу IQStocker Gen. Пользователь, оплативший подписку, получает лицензионный ключ с аппаратной привязкой (Session Lock на FastAPI-бэкенде). Встроенное Chrome-расширение связывает локальную сессию с нейросетями (например, Midjourney), перехватывая процессы: Pipeline локально распознает скачанные референсы через OpenAI Vision, инженерит из них богатые текстовые промпты (добавляя `--ar` и стили) и автоматически ставит в очередь генерации.

#### 4. Этап Разметки и Оформления (GPT-Vision-Batch & GPT_Video Vision)

На финальном этапе готовые сгенерированные файлы (тысячи картинок и видео) скармливаются десктопным программам с GUI на PySide6. Приложение собирает архивы и экономно отправляет их через OpenAI Batch API; нейросеть «просматривает» каждый файл и выдает выверенные до идеала метаданные (Title, Description, Keywords). Затем инструменты интегрируют ExifTool и FFmpeg для пофайлового инъекта: сгенерированные теги прописываются прямо в бинарную структуру EXIF для JPEG и метаданные MP4. Стокер получает 100% готовую к публикации на рыночных площадках базу файлов с идеальным поисковым SEO.

### Ключевые инженерные решения экосистемы

| Решение | Деталь | Инженерная сложность |
|---------|--------|---------------------|
| **Мультимодельный AI** | Бот работает с GPT-4, Claude и Gemini одновременно через единый `llm_service.py` | Абстракция над API разных провайдеров с unified response format |
| **Session Lock** | Лицензионный ключ привязан к fingerprint браузера, одновременная сессия невозможна | Распределённая блокировка: проверка на бэкенде + device fingerprinting в Extension |
| **Batch API оптимизация** | Вместо N запросов — один `.jsonl` файл через OpenAI Batch API (−50% стоимости) | Асинхронная оркестрация: сжатие → формирование батча → отправка → polling статуса → обработка результатов |
| **Мультипоточная обработка** | 5 параллельных потоков анализа + асинхронный батчинг | Контроль rate limits API, graceful degradation при ошибках, 2-часовой cooldown при исчерпании лимитов |
| **End-to-end метаданные** | AI → piexif (EXIF) / ExifTool / ffmpeg → готовый файл для стоков | Три разных инструмента записи для разных форматов (JPEG, RAW, MP4, MOV) |
| **FSM в Telegram** | Сложные цепочки диалогов: сбор CSV → настройка профиля → выбор AI-модели → результат | Finite State Machine на aiogram 3 с persistent state в Redis |
