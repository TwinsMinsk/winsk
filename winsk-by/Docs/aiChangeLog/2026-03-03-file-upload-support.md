# Добавление загрузки файлов в форму обратной связи

## Что было сделано:
1. **Типизация:** Обновлена схема валидации `contactSchema` в `src/types/contact.ts`, добавлено опциональное поле `file`.
2. **API (Backend):** Полностью переписан обработчик `POST` в `app/api/contact/route.ts`:
   - Теперь он поддерживает `multipart/form-data` вместо `JSON`.
   - Добавлена логика отправки файлов в Telegram через метод `sendDocument`.
   - Если файл не прикреплен, используется стандартный `sendMessage`.
   - Текст сообщения отправляется как `caption` (подпись к файлу).
3. **UI (Frontend):**
   - В компонент `ContactForm.tsx` добавлена кнопка "скрепка" (`Paperclip`) внутри текстового поля.
   - Реализован выбор файла, отображение его имени и размера, а также возможность удаления перед отправкой.
   - Метод `onSubmit` теперь формирует `FormData` для отправки на сервер.
   - Добавлен импорт иконок `Paperclip` и `X` из `lucide-react`.

## Зачем это было сделано:
По запросу пользователя для возможности прикрепления ТЗ, скриншотов или других материалов при обращении через форму на сайте.

## Отклонения от PRD/TechSpec:
Отклонений нет. Дизайн формы дополнен минималистичным индикатором выбранного файла в стиле Cyberpunk.

## Результаты проверки:
- Механика формирования `FormData` проверена.
- Интеграция с Telegram API реализована согласно официальной документации (метод `sendDocument`).
- Визуальное состояние формы (isSubmitting, success, error) корректно отрабатывает.
